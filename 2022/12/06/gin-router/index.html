【GIN学习笔记】Gin框架路由和路由组

发布于 2022-12-06 GIN学习笔记  golang gin 

Ⅰ 普通路由

router.GET("/index", func(ctx *gin.Context) {
    ctx.JSON(http.StatusOK, gin.H{
        "method": "GET",
    })
})

router.POST("/index", func(ctx *gin.Context) {
    ctx.JSON(http.StatusOK, gin.H{
        "method": "POST",
    })
})

router.DELETE("/index", func(ctx *gin.Context) {
    ctx.JSON(http.StatusOK, gin.H{
        "method": "DELETE",
    })
})

router.PUT("/index", func(ctx *gin.Context) {
    ctx.JSON(http.StatusOK, gin.H{
        "method": "PUT",
    })
})

也可以使用Any()函数，按Context.Request.Method的值来处理不同的请求方法：

router.Any("/any", func(ctx *gin.Context) {
    switch ctx.Request.Method {
        case http.MethodGet:
        ctx.JSON(http.StatusOK, gin.H{
            "method": "GET",
        })
        case http.MethodPost:
        ctx.JSON(http.StatusOK, gin.H{
            "method": "POST",
        })
        // ......
    }
})

对于没有配置处理函数的路由，可以指定一个默认的404方法：

router.NoRoute(func(ctx *gin.Context) {
    ctx.JSON(http.StatusNotFound, gin.H{
        "message": "404 not found",
    })
})

Ⅱ 路由组

通常将拥有共同URL前缀的路由划分到一个路由组下面，例如/video/index、/video/search、/video/detail就可以放到一个组内：

videoGroup := router.Group("/video")
{
    videoGroup.GET("/index", func(ctx *gin.Context) {
        ctx.JSON(http.StatusOK, gin.H{"msg": "video/index"})
    })
    videoGroup.GET("/detail", func(ctx *gin.Context) {
        ctx.JSON(http.StatusOK, gin.H{"msg": "video/detail"})
    })
    videoGroup.GET("/search", func(ctx *gin.Context) {
        ctx.JSON(http.StatusOK, gin.H{"msg": "video/search"})
    })
}

其中的{}仅方便代码阅读，对程序功能使不使用没有区别。

路由组支持嵌套操作；
